## 🎯 Основные изменения

### Исправлена обработка rate limit (429) от OpenRouter

Система теперь правильно обрабатывает rate limit ошибки от OpenRouter API и повторяет запросы вместо немедленного переключения на DeepL.

**Поведение до исправления:**
- OpenRouter возвращает 429 (rate limit)
- Система **сразу переключается на DeepL** без retry
- Пользователь не видит попыток повторной отправки

**Поведение после исправления:**
- OpenRouter возвращает 429 (rate limit)
- `openrouter.ts`: **3 retry** с ожиданием `Retry-After` секунд
- `translationPipeline.ts`: **5 дополнительных retry** на уровне pipeline
- Только после исчерпания всех попыток — остановка с ошибкой
- **DeepL НЕ вызывается** при rate limit (только при других ошибках)

## 🐛 Исправленные баги

### 1. translator.ts ловил RateLimitError и переключался на DeepL

**Проблема:**
```typescript
} catch (error: any) {
  // Ловил ВСЕ ошибки, включая RateLimitError
  console.warn('⚠️ [Translator] OpenRouter failed, falling back to DeepL');
  this.openrouterFailed = true;
  this.currentProvider = 'deepl';
  // ...
}
```

**Решение:**
```typescript
} catch (error: any) {
  // Если это RateLimitError — пробрасываем выше (не fallback)
  if (error.name === 'RateLimitError') {
    console.error('❌ [Translator] OpenRouter rate limit exhausted, rethrowing');
    throw error;
  }

  // Для остальных ошибок — fallback на DeepL
  console.warn('⚠️ [Translator] OpenRouter failed, falling back to DeepL');
  // ...
}
```

**Изменённые файлы:**
- `lib/translator.ts` — строки 198-201 (single translate)
- `lib/translator.ts` — строки 261-264 (batch translate)

### 2. Недостаточно логов в openrouter.ts

**Проблема:**
- При 429 был лог "Rate limit hit, waiting Xs..."
- Но не было лога **после успешного retry**
- Не было лога **после исчерпания всех попыток**

**Решение:**
```typescript
if (response.status === 429) {
  const retryAfter = parseInt(response.headers.get('Retry-After') || '60', 10);
  console.warn(`⚠️ [OpenRouter] Rate limit 429, waiting ${retryAfter}s before retry ${attempt}/${MAX_RETRIES}...`);

  if (attempt < MAX_RETRIES) {
    await this.sleep(retryAfter * 1000);
    console.log(`✅ [OpenRouter] Retry ${attempt}/${MAX_RETRIES} successful after ${retryAfter}s wait`);
    continue;
  } else {
    console.error(`❌ [OpenRouter] All ${MAX_RETRIES} retries exhausted, rate limit persists`);
    throw new RateLimitError(/*...*/);
  }
}
```

**Изменённый файл:**
- `lib/openrouter.ts` — строки 99-115

### 3. translationPipeline.ts не обрабатывал RateLimitError

**Проблема:**
```typescript
} catch (error) {
  if (error instanceof RateLimitError) {
    console.error(`[pipeline] Rate limit error: ${error.message}`);
    throw error; // Сразу бросал ошибку, без retry
  }
}
```

**Решение:**
Добавлен retry loop с максимум 5 попытками:

```typescript
const MAX_RATE_LIMIT_RETRIES = 5;
let rateLimitRetryCount = 0;

while (rateLimitRetryCount <= MAX_RATE_LIMIT_RETRIES) {
  try {
    const translated = await translator.translateBatch(uncachedTexts, { targetLang: 'RU' });
    // ... success, break
  } catch (error) {
    if (error instanceof RateLimitError) {
      rateLimitRetryCount++;

      if (rateLimitRetryCount > MAX_RATE_LIMIT_RETRIES) {
        throw new Error(`OpenRouter rate limit exceeded. Tried ${MAX_RATE_LIMIT_RETRIES} times.`);
      }

      const waitTime = error.retryAfter || 60;
      console.warn(`[pipeline] ⏳ OpenRouter rate limit. Waiting ${waitTime}s (attempt ${rateLimitRetryCount}/${MAX_RATE_LIMIT_RETRIES})...`);

      await new Promise(resolve => setTimeout(resolve, waitTime * 1000));
      continue; // Retry
    }
    throw error;
  }
}
```

**Изменённый файл:**
- `lib/translationPipeline.ts` — строки 146-195

## 🔄 Итоговая логика retry

| Уровень | Попытки | Поведение |
|---------|---------|-----------|
| `openrouter.ts` | 3 retry | Ждёт `Retry-After` секунд между попытками |
| `translator.ts` | 0 retry | Пробрасывает `RateLimitError` выше (не fallback) |
| `translationPipeline.ts` | 5 retry | Ждёт `retryAfter` секунд, повторяет весь batch |
| **Итого** | **до 18 попыток** | 3 × (1 + 5) = 18 максимум для одного batch |
