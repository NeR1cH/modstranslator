# Release v3.20.0 — Translation Resume & FragmentCache Quality Improvements

**Release Date:** 2026-05-10  
**Status:** Production Ready

---

## 🎯 Overview

This release introduces a robust translation resume system for handling rate limit errors and significantly improves FragmentCache quality by filtering out unreliable translations.

---

## ✨ New Features

### Translation Resume System
- **ProgressTracker** — new module for tracking translation progress
  - Saves progress to `.translation-cache/progress.json`
  - Automatically resumes translation after rate limit errors
  - File hash-based identification for reliable resume
  - Auto-expires after 24 hours
  - Integrated into translation pipeline

### OpenRouter Retry Logic
- **3 retry attempts** with exponential backoff
- **Retry-After header support** for rate limit responses
- **RateLimitError** class for proper error handling
- Nested try-catch for network errors
- Frontend receives rate limit events with resume instructions

---

## 🔧 Improvements

### FragmentCache Quality Fixes

**ИСПРАВЛЕНИЕ 1 — Remove Punctuation:**
- Strips punctuation from translations before saving
- Prevents fragments like "Золотой," or "руду." with trailing punctuation
- Cleaner cache entries for better reuse

**ИСПРАВЛЕНИЕ 2 — Filter Unknown Words:**
- Only saves known words (materials, prefixes, nouns with known gender)
- Blocks arbitrary words like "tree", "fluid", "speed", "use", "blocks"
- Reduces cache pollution from context-dependent translations

**ИСПРАВЛЕНИЕ 3 — Minimum Occurrence Count:**
- Requires minimum 2 occurrences before using fragment
- Increases reliability of cached translations
- Prevents one-off incorrect translations from propagating

### Results
- Cache reduced from 2,428 fragments (510KB) to 14 high-quality fragments
- 100% cache hit rate on repeated translations
- 13/14 fragments have ≥80% confidence
- No problematic words saved
