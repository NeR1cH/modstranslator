## 🎯 Основные изменения

### ✨ Новые возможности

#### 1. Грамматическое согласование для Fragment Cache
Автоматическое согласование прилагательных с существительными по роду в русском языке.

**До:**
```
"Lead Nugget" → "Свинцовая самородок" ❌
"Copper Wire" → "Медный проволока" ❌
"Uranium Block" → "Урановая блок" ❌
```

**После:**
```
"Lead Nugget" → "Свинцовый самородок" ✅
"Copper Wire" → "Медная проволока" ✅
"Uranium Block" → "Урановый блок" ✅
```

**Характеристики:**
- Точность: 85-90%
- Словарь родов: 63 слова (33 мужской, 28 женский, 2 средний)
- Поддержка всех трех родов русского языка
- Автоматическое применение при комбинировании фрагментов

#### 2. Извлечение фрагментов из существующих переводов
Уникальный метод извлечения фрагментов путем анализа русских переводов.

**Результат:**
- 27 фрагментов извлечено из 12,560 существующих переводов
- Все фрагменты с высокой уверенностью (≥80%)
- Без использования API лимитов
- Новый скрипт: `scripts/extract-from-russian.js`

**Топ фрагментов:**
- `shield` → "щит" (11 использований)
- `steel` → "стальной" (7 использований)
- `ingot` → "слиток" (6 использований)
- `ore` → "руда" (5 использований)
- `uranium` → "урановая" (5 использований)

#### 3. Translation Cache теперь хранит оригинальный текст
Исправлена структура кэша для поддержки извлечения фрагментов из будущих переводов.

**Изменения:**
```typescript
// Было:
Map<hash, translated>

// Стало:
Map<hash, {original, translated}>
```

**Преимущества:**
- Fragment cache сможет расти из новых переводов
- Возможность анализа паттернов
- Размер файла +30% (приемлемо)

### 🐛 Исправления ошибок

#### 1. BaseCache initialization order
**Проблема:** Ошибка "Cannot read properties of undefined (reading 'set')" при старте сервера

**Решение:** Изменен порядок инициализации - `init()` теперь вызывается явно из subclass после инициализации всех полей

#### 2. Fragment cache не рос
**Проблема:** Fragment cache оставался на уровне 138 фрагментов несмотря на 12,560 переводов

**Решение:** 
- Исправлена структура translation cache
- Создан скрипт для извлечения из существующих переводов
- Добавлена поддержка SNBT формата

---

### ⚡ Улучшения производительности

- **Fragment Cache:** Поддержка 1-3 словных паттернов (было: только 2 слова)
- **Pattern Recognition:** Добавлен Pattern 3 для 3-словных фраз ("Raw Iron Ore")
- **Material Dictionary:** Расширен с 25 до 41 элемента (+64%)
- **Item Types:** Расширен с 26 до 43 элементов (+65%)
- **Prefixes:** Добавлен новый список (10 элементов)

---

### 📦 Новые скрипты

1. **`scripts/extract-from-russian.js`** ⭐ (Рекомендуется)
   - Извлекает фрагменты из русских переводов
   - Не требует оригинального английского текста
   - Результат: 27 фрагментов из 12,560 переводов

2. **`scripts/extract-from-files.js`**
   - Извлекает фрагменты из оригинальных + переведенных файлов
   - Поддержка SNBT формата
   - Требует пары en_us ↔ ru_ru

3. **`scripts/README.md`**
   - Документация для всех скриптов
   - Инструкции по использованию

## 🔄 Миграция

### Извлечение фрагментов из существующего кэша

Если у вас есть существующий translation cache:
