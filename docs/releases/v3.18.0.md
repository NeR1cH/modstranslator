# Release v3.18.0 — Universal FragmentCache with Gender Agreement

**Date:** 2026-05-09  
**Status:** ✅ Released

---

## Overview

Major rewrite of FragmentCache to work with **any content type** (UI strings, descriptions, quests, items) and both translation providers (DeepL, OpenRouter). Added automatic Russian grammatical gender agreement system.

---

## Key Features

### 1. Universal Pattern Extraction ✨

FragmentCache now works with **any content**, not just Material+Item patterns:

**Before (v3.17.x):**
- Only extracted: Material + Item patterns ("Diamond Sword", "Iron Ore")
- UI strings ignored: "Enable Notifications" → no fragments

**After (v3.18.0):**
- Extracts from ANY content: UI, descriptions, quests, items
- 5 pattern extraction strategies:
  1. Exact phrase (2-4 words)
  2. Individual words (1:1 mapping)
  3. Single word
  4. Multiple → Single
  5. Sub-phrases (3+ words)

**Example:**
```
"Enable Notifications" → API → "Включить уведомления"
Fragments created:
  - "Enable Notifications" (phrase, confidence: 75)
  - "Enable" (word, confidence: 80)
  - "Notifications" (word, confidence: 80)

Next translation: "Disable Notifications"
  → FragmentCache HIT! No API call needed ✅
```

---

### 2. Gender Agreement System 🎯

Automatic Russian adjective-noun agreement:

**Three core functions:**

1. **`detectAdjectiveGender(word)`** — detects gender from endings:
   - `-ый, -ой, -ий` → masculine
   - `-ая, -яя` → feminine
   - `-ое, -ее` → neuter

2. **`normalizeToMasculine(adjective)`** — converts to base form:
   - "Свинцовая" → "Свинцовый"
   - "Медная" → "Медный"

3. **`applyGenderAgreement(adjective, gender)`** — generates correct form:
   - "Свинцовый" + feminine → "Свинцовая"
   - "Медный" + neuter → "Медное"

**How it works:**

```
Learning phase:
"Lead Ore" → API → "Свинцовая руда"
  Store: "Lead" → "Свинцовый" (normalized to masculine)
         "Ore" → "руда" (gender: feminine)

Usage phase:
"Zinc Ore" → FragmentCache:
  1. Find "Ore" → gender = feminine
  2. Find "Zinc" → "Цинковый" (isAdjective = true)
  3. Apply agreement: "Цинковый" + feminine → "Цинковая"
  4. Result: "Цинковая руда" ✅
  5. NO API CALL!
```

**Benefits:**
- ✅ Correct grammar: "Цинковая руда" (not "Цинковый руда")
- ✅ Memory efficient: stores only masculine form
- ✅ On-the-fly generation: variations generated when needed

### 3. Stop Words Filtering 🚫

47 common English words now excluded from fragments:

```typescript
STOP_WORDS = [
  'the', 'a', 'an', 'in', 'of', 'to', 'for', 'with', 'and', 'or',
  'but', 'is', 'are', 'was', 'were', 'be', 'been', 'being',
  'have', 'has', 'had', 'do', 'does', 'did', 'will', 'would',
  'should', 'could', 'may', 'might', 'must', 'can', 'at', 'by',
  'from', 'up', 'down', 'out', 'on', 'off', 'over', 'under',
  'again', 'further', 'then', 'once', 'here', 'there'
]
```

**Why:** These words don't translate consistently and create noise in cache.

## Technical Changes

### Fragment Interface

**Before:**
```typescript
interface Fragment {
  text: string;
  translation: string;
  context: 'prefix' | 'suffix' | 'standalone';
  gender?: 'masculine' | 'feminine' | 'neuter';
  count: number;
  confidence: number;
}
```

**After:**
```typescript
interface Fragment {
  text: string;
  translation: string;
  context: 'word' | 'phrase';           // CHANGED
  count: number;
  confidence: number;
  lastSeen: number;                     // NEW
  gender?: 'masculine' | 'feminine' | 'neuter';
  isAdjective?: boolean;                // NEW
}
```

## Known Issues

**Old cache entries:**

Some fragments created before v3.18.0 may have incorrect normalization:
- "Copper" → "Медные" (should be "Медный")
- "Brass" → "Латунные" (should be "Латунный")

**Impact:** Low — new translations will be correct, old entries will be overwritten over time.

**Fix options:**
1. Clear cache and start fresh
2. Run normalization script (not included)
3. Leave as-is (self-healing over time)
