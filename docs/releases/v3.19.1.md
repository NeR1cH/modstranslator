# Release v3.19.1 — FragmentCache Word-Only Mode

**Release Date:** 2026-05-09  
**Type:** Patch Release  
**Focus:** FragmentCache optimization

---

## 🎯 Overview

This release optimizes the FragmentCache system to save only individual words instead of phrases, improving fragment reusability across different contexts and reducing cache bloat.

---

## ✨ What's New

### FragmentCache Word-Only Mode

**Changed fragment extraction strategy:**
- ❌ Removed Pattern 1: Exact phrase saving
- ❌ Removed Pattern 3: Sub-phrases (2-3 word combinations)
- ✅ Kept Pattern 2: Individual words (1:1 mapping only)
- ✅ Kept single word case

**Example:**
```
Before: "Iron Sword" → saves full phrase "Iron Sword" → "Железный меч"
After:  "Iron Sword" → saves two fragments:
        - "Iron" → "Железный"
        - "Sword" → "меч"
```

**Benefits:**
- Better fragment reusability (e.g., "Iron" can be used in "Iron Pickaxe", "Iron Armor", etc.)
- Reduced cache size (no duplicate phrase variations)
- More flexible word-by-word assembly with gender agreement

---

## 🔧 Technical Changes

### Modified Files

**lib/fragmentCache.ts**
- `extractPatterns()` method simplified to save only individual words
- Removed phrase and sub-phrase extraction logic
- Kept gender detection and adjective normalization

## 📊 Impact

**Fragment Cache Behavior:**
- Saves only 1:1 word mappings when word counts match
- Saves single words regardless of translation word count
- Maintains gender agreement system for proper Russian grammar
- Reduces redundant phrase storage

---

## 🔄 Migration Notes

**Existing fragment caches:**
- Old phrase fragments will remain in cache but won't be added anymore
- New translations will only add individual word fragments
- No breaking changes to cache format or API
