### 🟠 Высокий приоритет (Should Have)

#### 2. Оптимизация производительности
**Статус:** ⏳ Не начато  
**Приоритет:** Высокий  
**Время:** 2-3 дня  
**Сложность:** Средняя

**Проблема:**
- Большие модпаки (1GB+) обрабатываются медленно
- Весь ZIP загружается в память
- Нет параллельной обработки JAR файлов
- Кэш не оптимизирован для больших объемов

**Решение:**

**2.1. Streaming обработка ZIP**
- Использовать `yauzl` вместо `JSZip` для streaming чтения
- Обрабатывать файлы по мере извлечения (не загружать весь ZIP в память)
- Писать результат в streaming ZIP (`yazl`)

**2.2. Параллельная обработка**
- Worker threads для обработки JAR файлов
- Пул воркеров (4-8 потоков)
- Очередь задач с приоритетами

**2.3. Оптимизация кэша**
- Индексация кэша (B-tree или hash index)
- Сжатие кэша (gzip)
- Периодическая очистка старых записей

**Польза:**
- Ускорение обработки в 2-3 раза
- Снижение потребления памяти на 50-70%
- Возможность обработки файлов 2GB+

**Файлы для изменения:**
- `lib/modpackProcessor.ts` - streaming ZIP
- `lib/jarProcessor.ts` - worker threads
- `lib/translationCache.ts` - индексация

**Зависимости:**
```json
{
  "dependencies": {
    "yauzl": "^2.10.0",
    "yazl": "^2.5.1"
  }
}
```
