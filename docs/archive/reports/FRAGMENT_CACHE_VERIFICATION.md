## 2. Извлечение паттернов (фрагментация) ✅

**Пример:** "Armor Status" → "Статус брони"
**Извлекает:** 
- "Armor Status" (phrase, confidence: 75)

**Результат:** ✅ Корректно

### Pattern 2: Individual words (1:1 mapping)
**Пример:** "Enable Notifications" → "Включить Уведомления"
**Извлекает:**
- "Enable Notifications" (phrase, 75)
- "Enable" (word, 80)
- "Notifications" (word, 80)

**Результат:** ✅ Корректно

### Pattern 2a: Single word
**Пример:** "Enable" → "Включить"
**Извлекает:**
- "Enable" (word, 85)

**Результат:** ✅ Корректно

### Pattern 2b: Multiple → Single
**Пример:** "Armor Status" → "Статус"
**Извлекает:**
- "Armor Status" (phrase, 70)

**Результат:** ✅ Корректно

### Pattern 3: Sub-phrases (3+ слова)
**Пример:** "Refined Copper Ingot" → "Очищенный медный слиток"
**Извлекает:**
- "Refined Copper Ingot" (phrase, 75)
- "Refined Copper" (phrase, 65)
- "Copper Ingot" (phrase, 65)
- "Refined" (word, 80)
- "Copper" (word, 80)
- "Ingot" (word, 80)

**Потенциальная проблема:** Предполагает 1:1 позиционное соответствие слов
**Защита:** Низкий confidence (65) означает низкий приоритет использования
**Результат:** ⚠️ Может создавать шум, но защищён низким confidence

## 3. Согласование по роду ✅

### detectAdjectiveGender()
**Проверено:** Определение рода по окончанию
- `-ый, -ой, -ий` → masculine ✅
- `-ая, -яя` → feminine ✅
- `-ое, -ее` → neuter ✅

**Результат:** ✅ Корректно

### normalizeToMasculine()
**Проверено:** Нормализация к мужскому роду
- "Свинцовая" → "Свинцовый" ✅
- "Медная" → "Медный" ✅

**Результат:** ✅ Корректно

### applyGenderAgreement()
**Проверено:** Применение согласования
- masculine: "Свинцовый" → "Свинцовый" ✅
- feminine: "Свинцовый" → "Свинцовая" ✅
- neuter: "Свинцовый" → "Свинцовое" ✅

**Результат:** ✅ Корректно

### Реальный сценарий
```
Input: "Lead Ore" → "Свинцовая руда"
Learn: "Lead" (adj, stored as "Свинцовый"), "Ore" (fem)

Use: "Zinc Ore"
1. Find "Ore" → gender=feminine
2. Find "Zinc" → isAdjective=true
3. Apply agreement: "Цинковый" → "Цинковая"
4. Result: "Цинковая руда" ✅
```

**Результат:** ✅ Согласование работает правильно

## 4. Процесс learn() (обучение) ✅

### Шаг 1: Извлечение паттернов
**Проверено:** `extractPatterns(original, translated)`
**Результат:** ✅ Возвращает массив фрагментов

### Шаг 2: Обработка фрагментов

**Если фрагмент существует:**
- `count++` ✅
- `lastSeen = now` ✅
- Если перевод совпадает: `confidence += 3` (max 100) ✅
- Если перевод отличается: `confidence -= 10` (min 30) ✅
- Обновление `gender`/`isAdjective` если не установлены ✅

**Если фрагмент новый:**
- Создание нового фрагмента со всеми полями ✅

**Результат:** ✅ Логика обучения корректна

## 5. Процесс tryTranslate() (использование) ✅

### Шаг 1: Exact match
**Проверено:**
- Нормализация текста (lowercase)
- Поиск в кэше
- Проверка `confidence >= 70`

**Результат:** ✅ Корректно

### Шаг 2: Word-by-word translation

**First pass — поиск рода существительного:**
- Идёт с конца (справа налево) ✅
- Ищет `fragment.gender` ✅
- Ищет в `NOUN_GENDERS` ✅
- Останавливается на первом найденном ✅

**Примечание:** В английском существительное обычно последнее, поэтому поиск справа налево правильный.

**Second pass — перевод с согласованием:**
- Для каждого слова ищет фрагмент ✅
- Проверка `confidence >= 60` ✅
- Если `isAdjective && nounGender`: применяет согласование ✅
- Если хоть одно слово не найдено: `return null` ✅

**Проверка средней уверенности:**
- `avgConfidence >= 70` ✅

**Результат:** ✅ Логика использования корректна
