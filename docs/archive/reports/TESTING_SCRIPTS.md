# Скрипты автоматического тестирования

Этот проект включает несколько batch-скриптов для автоматизации тестирования и запуска сервера.

## 📋 Доступные скрипты

### 🌟 `dev-menu.bat` - Универсальное меню (РЕКОМЕНДУЕТСЯ)
Объединяет функционал всех скриптов в одном удобном меню.

**Использование:**
```bash
scripts\dev-menu.bat
```

**Меню опций:**
```
========================================
  MOD_TRANSLATOR - Test & Dev Menu
========================================

Choose an option:

  1. Run tests only
  2. Start dev server (no tests)
  3. Start dev server with test check
  4. Start dev server with auto-fix (3 attempts)
  5. Exit
```

**Что делает каждая опция:**

1. **Run tests only** - Запускает тесты с покрытием, показывает результат, возвращает в меню
2. **Start dev server (no tests)** - Сразу запускает dev-сервер без проверок
3. **Start dev server with test check** - Запускает тесты, при успехе → dev-сервер. При ошибке предлагает:
   - Повторить тесты
   - Запустить сервер всё равно
   - Вернуться в меню
4. **Start dev server with auto-fix** - Умный режим с 3 попытками:
   - Запускает тесты
   - При ошибке: очищает кэш Jest, переустанавливает зависимости
   - Повторяет до 3 раз
   - При неудаче предлагает открыть лог или запустить сервер
5. **Exit** - Выход из скрипта

---

### 1. `run-tests.bat` - Простой запуск тестов
Запускает все тесты с отчетом о покрытии кода.

**Использование:**
```bash
run-tests.bat
```

**Что делает:**
- Запускает `npm test -- --coverage`
- Показывает результаты тестов
- Выводит SUCCESS или FAILED

---

### 2. `start-with-tests.bat` - Запуск с проверкой
Запускает тесты перед стартом dev-сервера. Если тесты не проходят, предлагает исправить и повторить.

**Использование:**
```bash
start-with-tests.bat
```

**Что делает:**
1. Запускает все тесты
2. Если тесты прошли → запускает `npm run dev`
3. Если тесты не прошли → предлагает:
   - Вариант 1: Повторить тесты (после исправления)
   - Вариант 2: Выйти

**Пример работы:**
```
========================================
Pre-flight Check: Running Tests
========================================

Running tests...

[FAILED] Tests failed!

Please fix the errors and choose an option:
  1. Retry tests
  2. Exit

Enter your choice (1 or 2): 1
```

---

### 3. `start-with-auto-fix.bat` - Запуск с авто-исправлением
Самый умный скрипт! Пытается автоматически исправить проблемы и повторяет тесты до 3 раз.

**Использование:**
```bash
start-with-auto-fix.bat
```

**Что делает:**
1. Запускает тесты (попытка 1/3)
2. Если тесты не прошли:
   - Очищает кэш Jest
   - Переустанавливает зависимости
   - Повторяет тесты (попытка 2/3)
3. Если после 3 попыток тесты не прошли:
   - Сохраняет лог ошибок в `test-output.log`
   - Предлагает открыть лог для анализа
4. Если тесты прошли → запускает `npm run dev`

**Пример работы:**
```
========================================
MOD_TRANSLATOR - Auto-Fix Start Script
========================================

[Attempt 1/3] Running tests...

[FAILED] Tests failed on attempt 1

Attempting auto-fix...
[Auto-fix] Clearing Jest cache...
[Auto-fix] Reinstalling dependencies...

Retrying tests...

[Attempt 2/3] Running tests...

[SUCCESS] All tests passed!
Starting development server...
```

## 🚀 NPM команды

Также доступны через npm:

```bash
# Обычный запуск dev-сервера
npm run dev

# Запуск с предварительным тестированием
npm run dev:safe

# Только тесты
npm test

# Тесты с покрытием
npm run test:coverage

# Тесты в режиме наблюдения
npm run test:watch
```

## 💡 Рекомендации

**Для ежедневной разработки:**
- Используйте `scripts\dev-menu.bat` - универсальное меню со всеми опциями
- Или `npm run dev:safe` для быстрого запуска с проверкой тестов

**Для быстрой проверки:**
- Используйте `scripts\run-tests.bat` или `npm test`

**При проблемах с тестами:**
- Используйте опцию 4 в `dev-menu.bat` (auto-fix)
- Или `scripts\start-with-auto-fix.bat`
- Скрипт попытается автоматически исправить типичные проблемы

**Для CI/CD:**
- Используйте `npm test` в pipeline
- Все скрипты возвращают правильные exit codes (0 = успех, 1 = ошибка)

## 🔧 Устранение проблем

### Тесты не запускаются
```bash
# Очистить кэш Jest
npm test -- --clearCache

# Переустановить зависимости
npm install
```

### Порт 3000 занят
```bash
# Используйте dev:clean для автоматической очистки портов
npm run dev:clean
```

### Тесты падают после обновления кода
1. Запустите `start-with-auto-fix.bat`
2. Если не помогло, проверьте `test-output.log`
3. Исправьте ошибки вручную
4. Повторите попытку

---

## 📝 Примечания

- Все batch-скрипты работают только на Windows
- Для Linux/Mac создайте аналогичные shell-скрипты (.sh)
- Логи сохраняются в `test-output.log` (автоматически удаляется при успехе)
- Скрипты используют `enabledelayedexpansion` для корректной работы переменных в циклах
