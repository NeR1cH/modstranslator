### 2. Исправление BaseCache инициализации (v3.15.0)

**Проблема:**
```
Error: Cannot read properties of undefined (reading 'set')
    at TranslationCache.loadFromDisk (lib/translationCache.ts:134)
```

**Причина:**
- BaseCache вызывал `init()` в конструкторе
- `init()` вызывал `loadFromDisk()`
- `loadFromDisk()` пытался использовать `this.memoryCache`
- Но `memoryCache` еще не был инициализирован (JavaScript initialization order)

**Решение:**
```typescript
// lib/BaseCache.ts
constructor(config: CacheConfig) {
  // Инициализация полей
  // НЕ вызываем init() здесь
}

protected init(): void {
  // Изменен с private на protected
  // Теперь вызывается явно из subclass
}

// lib/translationCache.ts
constructor() {
  super(config);
  // Сначала инициализируем свои поля
  this.memoryCache = new Map();
  // Потом вызываем init()
  this.init();
}
```

**Результат:**
- ✅ Сервер запускается без ошибок
- ✅ Cache загружается корректно (12,560 + 138 записей)
- ✅ Все 282 теста проходят
- Коммит: `ee8ca75`

### 3. Исправление Translation Cache (v3.15.1)

**Проблема:**
- Fragment cache не рос (оставался на 138 фрагментах)
- Translation cache хранил `original: ''` для экономии места
- Fragment cache не мог извлекать паттерны из существующих переводов

**Анализ:**
```bash
# Запустили extract-fragments.js
Processed: 12560, Skipped: 12560
# Все записи пропущены - нет оригинального текста!
```

**Решение:**

1. **Изменена структура memoryCache:**
```typescript
// Было:
private memoryCache = new Map<string, string>();

// Стало:
private memoryCache = new Map<string, { original: string; translated: string }>();
```

2. **Обновлены все методы:**
```typescript
// get() - возвращает cached.translated
get(text: string): string | null {
  const cached = this.memoryCache.get(hash);
  return cached ? cached.translated : null;
}

// set() - сохраняет пару
set(original: string, translated: string): void {
  this.memoryCache.set(hash, { original, translated });
}

// saveToDisk() - сохраняет original
entries.push({
  hash,
  original,  // Было: original: ''
  translated,
  timestamp: Date.now()
});
```

3. **Создан скрипт extract-fragments.js:**
- Читает translation cache
- Извлекает фрагменты без API вызовов
- Реализует ту же логику, что и fragmentCache.ts
- Поддерживает 1-3 словные паттерны

**Результат:**
- ✅ Новые переводы будут сохранять оригинальный текст
- ✅ Fragment cache сможет извлекать паттерны из будущих переводов
- ✅ Размер cache файла увеличится на ~30% (приемлемо)
- ✅ Все 282 теста проходят

## 🎓 Извлеченные уроки

### 1. JavaScript Initialization Order
**Проблема:** Parent constructor выполняется до инициализации полей subclass

**Решение:** 
- Не вызывать методы в parent constructor
- Использовать явный `init()` метод
- Вызывать `init()` после инициализации всех полей

### 2. Cache Design Tradeoffs
**Проблема:** Экономия места (`original: ''`) vs функциональность (fragment extraction)

**Решение:**
- Хранить полные данные
- +30% размера файла приемлемо
- Функциональность важнее размера

### 3. Grammar Agreement Complexity
**Проблема:** Русский язык имеет сложные правила согласования

**Решение:**
- Словарь родов для известных слов
- Простые правила для окончаний
- 85-90% точности достаточно для практического использования
