### 2. ✅ Улучшение Fragment Cache (v3.14.0)

**Цель:** Увеличить эффективность fragment cache в 15-20 раз

**Проблема:**
- 12,560 записей в translation cache
- Только 138 фрагментов (соотношение 91:1)
- Fragment cache hit rate ~5%

**Решение:**

1. **Расширены списки (+64% и +65%):**
   - MATERIALS: 25 → 41 элемент
   - ITEM_TYPES: 26 → 43 элемента
   - PREFIXES: 0 → 10 элементов (новый)

2. **Поддержка 1-3 слов в переводе:**
   - Было: только ровно 2 слова
   - Стало: 1, 2 или 3 слова

3. **Pattern 3 для 3-словных фраз:**
   - "Raw Iron Ore" → ["raw", "iron", "ore"]
   - "Crushed Gold Dust" → ["crushed", "gold", "dust"]
   - "Refined Steel Ingot" → ["refined", "steel", "ingot"]

4. **Обновлен detectPatterns():**
   - Приоритет: 3-словные паттерны → 2-словные
   - Извлекаются все распознанные части

**Результат:**
- ✅ Ожидаемое увеличение фрагментов: 138 → 2,000-3,000 (15-20x)
- ✅ Ожидаемый hit rate: ~5% → 30-40% (6-8x)
- ✅ Ожидаемая экономия API: +25-35%
- ✅ Добавлено 11 новых тестов (всего 276)
- ✅ Все тесты проходят

4. **Обновлены extractPatterns() и learn():**
   - Сохраняют род при извлечении фрагментов
   - Переиспользуют род при комбинировании

**Результат:**
