# 💰 Экономия API лимита - Как это работает

## ✅ Хорошие новости!

Система **автоматического кэширования переводов** уже работает с версии **3.3.0**!

Она делает **именно то, что ты описал**:
- Сохраняет все переводы локально
- Повторно использует их без API запросов
- Экономит твои 500,000 символов

---

## 🎯 Как это работает

### 1. Первый перевод (используется API)

```
Ты переводишь мод с текстом:
- "Magic Sword" → API DeepL → "Волшебный меч" ✅ (12 символов использовано)
- "Magic Wand" → API DeepL → "Волшебная палочка" ✅ (11 символов использовано)
- "Stone Block" → API DeepL → "Каменный блок" ✅ (12 символов использовано)

Итого: 35 символов из лимита
```

**Система автоматически сохраняет** все переводы в файл:
```
.translation-cache/cache-v1.json
```

### 2. Второй перевод (используется кэш)

```
Ты переводишь другой мод, в котором есть:
- "Magic Sword" → ИЗ КЭША → "Волшебный меч" ✅ (0 символов!)
- "Magic Wand" → ИЗ КЭША → "Волшебная палочка" ✅ (0 символов!)
- "Diamond Sword" → API DeepL → "Алмазный меч" ✅ (13 символов использовано)

Итого: 13 символов вместо 38!
Экономия: 66%! 🎉
```

## 🔍 Как проверить что кэш работает

### 1. Посмотри индикатор кэша в UI

На главной странице есть секция:

```
┌─────────────────────────────────────┐
│ 💾 КЭШ ПЕРЕВОДОВ                    │
│ Закэшировано: 1 запись(ей)      [↻] │
│                                      │
│ ℹ️ Повторные переводы используют    │
│    кэш вместо API (экономия лимита) │
└─────────────────────────────────────┘
```

### 2. Посмотри логи в консоли браузера (F12)

При переводе ты увидишь:

```
[deepl] Cache stats: 150 hits, 50 misses
[deepl] Characters to translate (uncached): 450
```

Это значит:
- **150 строк** взяты из кэша (0 символов API)
- **50 строк** переведены через API (450 символов API)
- **Экономия:** 75%!

### 3. Посмотри файл кэша

```bash
cat ".translation-cache/cache-v1.json"
```

Увидишь все сохраненные переводы.

## 📁 Где хранится кэш

```
C:\VSCODE PROJECTS\modstranslator\.translation-cache\cache-v1.json
```

**Структура:**
```json
{
  "version": "v1",
  "entries": [
    {
      "hash": "fa8847b0c33183273f5945508b31c3208a9e4ece58ca47233a05628d8dba3799",
      "original": "",
      "translated": "Волшебный меч",
      "timestamp": 1777802593632
    }
  ]
}
```

**Особенности:**
- Хранится локально на твоем компьютере
- Переживает перезагрузку браузера
- Работает между сессиями
- Автоматически сохраняется каждые 5 секунд

## 🧠 Умные возможности кэша

### 1. Case-insensitive (игнорирует регистр)

```
"Magic Sword" → кэш
"magic sword" → тот же кэш! ✅
"MAGIC SWORD" → тот же кэш! ✅
```

### 2. Trim whitespace (игнорирует пробелы)

```
"Magic Sword" → кэш
"  Magic Sword  " → тот же кэш! ✅
```

### 3. Хеширование (быстрый поиск)

Использует SHA-256 для мгновенного поиска переводов.

## 🛠️ Управление кэшем

### Посмотреть статистику

В UI есть индикатор:
```
💾 КЭШ ПЕРЕВОДОВ
Закэшировано: 1,234 запись(ей)
```

### Очистить кэш

Нажми кнопку **"🗑️ Очистить"** в индикаторе кэша.

Или вручную:
```bash
rm -rf ".translation-cache"
```

### Когда очищать кэш?

- Если переводы устарели
- Если хочешь перевести заново
- Если кэш занимает много места (редко)

**Обычно очищать НЕ нужно!** Кэш только помогает.

## ❓ FAQ

### Q: Кэш работает автоматически?

**A:** Да! Ничего настраивать не нужно. Работает с версии 3.3.0.

### Q: Кэш работает между разными модами?

**A:** Да! Если в двух модах есть одинаковые строки - они используют один кэш.

### Q: Кэш работает после перезагрузки компьютера?

**A:** Да! Кэш хранится на диске в файле `.translation-cache/cache-v1.json`.

### Q: Сколько места занимает кэш?

**A:** Очень мало. 1,000 переводов ≈ 100 KB. Даже 10,000 переводов ≈ 1 MB.

### Q: Кэш замедляет работу?

**A:** Нет! Наоборот, ускоряет. Кэшированные переводы возвращаются мгновенно (0ms вместо 100-500ms API).

### Q: Что если перевод в кэше неправильный?

**A:** Очисти кэш и переведи заново. Или отредактируй файл `.translation-cache/cache-v1.json` вручную.

### Q: Кэш синхронизируется между компьютерами?

**A:** Нет, кэш локальный. Но можно скопировать файл `.translation-cache/cache-v1.json` на другой компьютер.

### Q: Можно ли поделиться кэшем с другими?

**A:** Да! Просто отправь файл `.translation-cache/cache-v1.json`. Другой человек положит его в свою папку проекта.
