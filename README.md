# MOD_TRANSLATOR — Minecraft Localization Engine

Автоматический переводчик модов Minecraft на русский язык через DeepL API.

**Максимальный размер файла: 1000 MB**

## Поддерживаемые форматы

| Формат | Версия MC | Описание |
|--------|-----------|----------|
| `.jar` | 1.13+ | Современные моды — ищет `assets/*/lang/en_us.json`, добавляет `ru_ru.json` |
| `.jar` | до 1.13 | Старые моды — ищет `assets/*/lang/en_US.lang`, добавляет `ru_ru.lang` |
| `.zip` | любая | Модпаки — сканирует все файлы внутри, переводит поддерживаемые форматы |
| `.json` | любая | Отдельный lang-файл или nested JSON (квесты, диалоги, книги Patchouli) |
| `.lang` | любая | Отдельный lang-файл старого формата |
| `.snbt` | любая | Квесты FTB Quests / Better Questing |
| `.toml` | любая | Конфигурационные файлы Forge |
| `.cfg` | любая | Конфигурационные файлы старых версий Forge |
| `.xml` | любая | XML диалоги и субтитры |
| `.txt` | любая | Текстовые файлы субтитров и скриптов |

## Возможности

- ✅ **Автоматическое определение формата** — система сама распознает тип файла
- ✅ **Пакетная обработка** — загрузите несколько модов или целый модпак одновременно
- ✅ **Сохранение форматирования** — поддерживает коды форматирования Minecraft (§6, %s, {0})
- ✅ **Глубокий анализ модпаков** — сканирует все файлы внутри ZIP-архива
- ✅ **Поддержка квестов** — переводит FTB Quests, Better Questing, Patchouli книги
- ✅ **Конфигурации** — переводит конфиги модов (.toml, .cfg)
- ✅ **Диалоги и катсцены** — поддержка XML и текстовых файлов
- ✅ **Безопасность** — API ключ хранится только на сервере

## Быстрый старт

### 1. Установи Node.js
Скачай с https://nodejs.org (версия 18 или выше)

Проверь установку:
```bash
node --version  # должно быть v18.x.x или выше
npm --version
```

### 2. Получи DeepL API ключ
Зарегистрируйся на https://www.deepl.com/pro-api

- **Бесплатный план** — 500 000 символов/месяц (ключ оканчивается на `:fx`)
- **Pro план** — без ограничений (ключ оканчивается на другими символами)

### 3. Настрой проект

```bash
# Распакуй архив и открой папку в терминале
cd minecraft-mod-translator

# Установи зависимости
npm install

# Создай файл с ключом (скопируй из .env.example)
copy .env.example .env        # Windows
cp .env.example .env          # Mac/Linux

# Открой .env и вставь свой ключ DeepL
DEEPL_API_KEY=твой_ключ_здесь
```

### 4. Запусти

```bash
npm run dev
```

Открой браузер: **http://localhost:3000**

## Как пользоваться

### Перевод отдельного мода (.jar)

1. Перетащи `.jar` файл мода в зону загрузки (или кликни для выбора)
2. Подожди анализа — увидишь сколько языковых файлов и строк найдено
3. Нажми **ЗАПУСТИТЬ ПЕРЕВОД**
4. После завершения нажми **СКАЧАТЬ АРХИВ**
5. Внутри архива — готовый `.jar` файл с добавленными `ru_ru.json`/`ru_ru.lang`

### Перевод модпака (.zip)

1. Перетащи `.zip` архив модпака в зону загрузки
2. Система просканирует все файлы внутри:
   - Языковые файлы модов (`en_us.json` → `ru_ru.json`)
   - Квесты (.snbt)
   - Конфигурации (.toml, .cfg)
   - Книги Patchouli (.json)
   - Диалоги (.xml, .txt)
3. Нажми **ЗАПУСТИТЬ ПЕРЕВОД**
4. Скачай готовый архив с переведённым модпаком

### Перевод отдельных файлов

Поддерживается загрузка:
- `.json` — языковые файлы, квесты, диалоги
- `.lang` — старые языковые файлы
- `.snbt` — квесты FTB/Better Questing
- `.toml` / `.cfg` — конфигурации
- `.xml` — диалоги, субтитры
- `.txt` — текстовые файлы

## Детали работы

### Анализ файлов

При загрузке файла система:
1. Определяет формат по расширению
2. Для `.jar` — извлекает языковые файлы из `assets/*/lang/`
3. Для `.zip` — сканирует всю структуру архива
4. Для остальных — парсит содержимое напрямую
5. Подсчитывает количество строк для перевода
6. Показывает статистику пользователю

### Процесс перевода

1. Извлечение текстовых значений из оригинальных файлов
2. Группировка в батчи по 50 строк (оптимизация API запросов)
3. Отправка в DeepL API с параметрами:
   - `target_lang=RU` — целевой язык
   - `source_lang=EN` — исходный язык
   - `tag_handling=xml` — сохранение тегов форматирования
   - `ignore_tags=keep` — защита специальных символов
4. Вставка переведённых значений обратно в файлы
5. Генерация выходного архива

### Поддерживаемые пути в JAR

Система ищет файлы по паттернам:
- `assets/*/lang/en_us.json` → `ru_ru.json`
- `assets/*/lang/en_US.lang` → `ru_ru.lang`
- `assets/*/lang/en.json` → `ru_ru.json`
- `assets/*/lang/en.lang` → `ru_ru.lang`

### Обработка модпаков

В ZIP-архивах система обрабатывает:
- Все языковые файлы модов внутри архива
- FTB Quests: `*.snbt` файлы
- Конфигурации: `*.toml`, `*.cfg`
- Patchouli книги: JSON файлы с ключевыми словами (patchouli, book, guide)
- Квесты и диалоги: JSON файлы с ключевыми словами (quest, dialogue, story)
- XML диалоги: файлы с ключевыми словами (dialogue, subtitle, cutscene)
- Текстовые субтитры: файлы с ключевыми словами (subtitle, script)

Пропускаются:
- Бинарные файлы (`.class`, `.png`, `.jpg`, и т.д.)
- Вложенные архивы (`.jar`, `.zip` внутри модпака)
- Системные папки (`META-INF`, `node_modules`, `.git`)

## Безопасность

`DEEPL_API_KEY` хранится **только** в `.env` на сервере.
Переменная **не** имеет префикс `NEXT_PUBLIC_` — браузер никогда её не видит.
Файл `.env` добавлен в `.gitignore`.

API запросы выполняются исключительно на стороне сервера через Next.js API Routes.

## Troubleshooting

### Ошибка "DEEPL_API_KEY не задан"
- Проверь наличие файла `.env` в корне проекта
- Убедись, что ключ скопирован без лишних пробелов
- Перезапусти сервер разработки после изменения `.env`

### Ошибка "Нет английских lang файлов в JAR"
- Мод может не содержать локализацию
- Языковые файлы могут быть в нестандартной папке
- Файлы могут быть обфусцированы внутри `.class`

### Ошибка 429 (Too Many Requests)
- Превышен лимит запросов к DeepL API
- Подождите немного, система автоматически повторит попытку

### Файл не загружается
- Проверьте размер файла (максимум 1000 MB)
- Убедитесь, что расширение файла поддерживается
- Проверьте консоль браузера на ошибки
