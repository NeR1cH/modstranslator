# 🟩 MOD_TRANSLATOR — Minecraft Localization Engine

> Автоматический переводчик модов и модпаков Minecraft с английского на русский язык через DeepL API.  
> Работает прямо в браузере — просто перетащи файл, нажми кнопку, скачай готовый результат.

## ✨ Возможности

| Функция | Описание |
|---------|----------|
| 🤖 **Авто-определение формата** | Система сама распознаёт тип файла по расширению и содержимому |
| 📦 **Поддержка модпаков** | Загрузи ZIP-архив целого модпака — переведётся всё внутри |
| 🫙 **Обработка JAR** | Поддержка как современных модов (1.13+), так и старых (до 1.13) |
| 🗂️ **Батчевая обработка** | Переводит строки группами по 50 для оптимизации API-запросов |

| ⚙️ **Конфигурации** | Переводит текстовые поля в `.toml` и `.cfg` файлах |
| 🔐 **Безопасность ключа** | API ключ DeepL хранится только на сервере, браузер его не видит |
| 💾 **Размер до 1000 MB** | Поддерживает даже тяжёлые модпаки |

## 📁 Поддерживаемые форматы

| Формат | Версия MC | Описание |
|--------|-----------|----------|
| `.jar` | 1.13+ | Современные моды — ищет `assets/*/lang/en_us.json`, добавляет `ru_ru.json` |
| `.jar` | до 1.13 | Старые моды — ищет `assets/*/lang/en_US.lang`, добавляет `ru_ru.lang` |
| `.zip` | любая | Модпаки — сканирует все файлы внутри, переводит поддерживаемые форматы |
| `.json` | любая | Плоский lang-файл ИЛИ вложенный JSON (квесты, Patchouli-книги, диалоги) |
| `.lang` | любая | Старый формат языковых файлов (`key=value`) |
| `.snbt` | любая | Квесты FTB Quests / Better Questing |
| `.toml` | любая | Конфиги Forge-модов |
| `.cfg` | любая | Конфиги старых версий Forge |
| `.xml` | любая | XML диалоги и субтитры |
| `.txt` | любая | Текстовые файлы субтитров и скриптов |

### Что пропускается автоматически

При обработке модпаков система **пропускает**:
- Бинарные файлы (`.class`, `.png`, `.jpg`, `.ogg`, `.mp3`, ...)
- Вложенные архивы (`.jar`, `.zip` внутри ZIP)
- Системные папки (`META-INF`, `node_modules`, `.git`)
- Файлы без переводимого текста (менее 2 символов, без латиницы)

## 🚀 Быстрый старт

### 1. Установи Node.js

Скачай с [nodejs.org](https://nodejs.org) — **версия 18 или выше**.

```bash
node --version   # должно быть v18.x.x или выше
npm --version
```

### 2. Получи DeepL API ключ

Зарегистрируйся на [deepl.com/pro-api](https://www.deepl.com/pro-api):

- **Бесплатный план** — 500 000 символов/месяц. Ключ оканчивается на `:fx`
- **Pro план** — без ограничений по символам

### 3. Установи зависимости и настрой окружение

```bash
# Перейди в папку проекта
cd minecraft-mod-translator

# Установи зависимости
npm install

# Создай файл с ключом из примера
cp .env.example .env       # Mac / Linux
copy .env.example .env     # Windows
```

Открой `.env` и вставь свой ключ:

```env
DEEPL_API_KEY=твой_ключ_deepl_здесь
```

### 4. Запусти

```bash
npm run dev
```

Открой браузер: **http://localhost:3000**

## 🎮 Как пользоваться

### Перевод мода (`.jar`)

1. Перетащи `.jar` файл в зону загрузки (или кликни для выбора)
2. Система проанализирует JAR: покажет количество языковых файлов и строк
3. Нажми **ЗАПУСТИТЬ ПЕРЕВОД** — прогресс отображается в реальном времени
4. После завершения нажми **СКАЧАТЬ** — получишь исходный JAR с добавленными `ru_ru.json` / `ru_ru.lang`

> Оригинальные английские файлы остаются нетронутыми — добавляются только русские

### Перевод модпака (`.zip`)

1. Перетащи `.zip` архив модпака
2. Система просканирует всё содержимое:
   - Языковые файлы всех модов внутри
   - SNBT-квесты FTB / Better Questing
   - Конфигурации Forge (`*.toml`, `*.cfg`)
   - Patchouli-книги и квесты (вложенный JSON)
   - XML диалоги и текстовые субтитры
3. Нажми **ЗАПУСТИТЬ ПЕРЕВОД**
4. Скачай готовый архив — структура оригинального модпака сохранена

### Перевод отдельных файлов

Поддерживается загрузка любого из следующих типов напрямую:
`.json`, `.lang`, `.snbt`, `.toml`, `.cfg`, `.xml`, `.txt`

## 🔬 Архитектура и детали работы

### Поток данных

```
Пользователь загружает файл (base64 в JSON)
        ↓
POST /api/analyze  →  возвращает статистику (строк, файлов, режим)
        ↓
POST /api/translate  →  основная обработка
        ↓
  ┌─────────────────────────────────────────┐
  │  Роутер по расширению                   │
  │  .zip → translateModpack()              │
  │  .jar → extractLangFiles()              │
  │         translateLangFiles()            │
  │         repackJar()                     │
  │  прочие → getStandaloneHandler()        │
  └─────────────────────────────────────────┘
        ↓
  translateTexts() — батчинг по 50 строк
        ↓
  DeepL API (Free или Pro — определяется по ключу)
        ↓
  Результат: base64 готового файла
        ↓
Пользователь скачивает архив / файл
```

### Модуль `deepl.ts` — клиент DeepL API

- **Авто-определение плана**: ключи, оканчивающиеся на `:fx` → Free endpoint, остальные → Pro endpoint
- **Батчинг**: не более 50 строк за один запрос к API (ограничение DeepL)
- **Retry логика**: при HTTP 429 или 5xx — до 3 повторных попыток с нарастающей задержкой
- **Параметры запроса**:
  - `target_lang=RU`, `source_lang=EN`
  - `tag_handling=xml` + `ignore_tags=keep` — защита кодов форматирования Minecraft

### Модуль `jarProcessor.ts` — обработка JAR

JAR-файлы являются ZIP-архивами. Система:
1. Открывает JAR через JSZip
2. Сканирует все пути по паттернам:
   - `assets/*/lang/en_us.json` → добавляет `ru_ru.json`
   - `assets/*/lang/en_US.lang` → добавляет `ru_ru.lang`
   - `assets/*/lang/en.json` / `en.lang` — тоже поддерживается
3. Парсит найденные файлы, собирает строки для перевода
4. После перевода перепаковывает JAR, добавляя русские файлы
5. Оригинальные английские файлы **сохраняются** — мод продолжает работать без русского языка

### Модуль `modpackProcessor.ts` — обработка модпаков

Стратегия определения для каждого файла в ZIP:

| Паттерн | Стратегия |
|---------|-----------|
| `assets/*/lang/en*` | `lang_json_or_lang` |
| `*.snbt` | `snbt` |
| `*.toml` | `toml` |
| `*.cfg` | `cfg` |
| `*.json` + `patchouli/quest/dialogue/book/story/...` в пути | `nested_json` |
| `*.json` + `/lang/` в пути | `lang_json` |
| `*.xml` + `dialogue/subtitle/cutscene` в пути | `xml` |
| `*.txt` + `subtitle/script/dialogue` в пути | `txt` |

Для языковых файлов модов: добавляется русская версия (`ru_ru`), оригинал сохраняется.  
Для квестов и конфигов: файл перезаписывается переведённой версией на том же месте.

### Модуль `langParsers.ts` — парсеры форматов

Каждый формат имеет пару функций: `parse*` и `rebuild*`:

- **`parseJsonLang` / `rebuildJsonLang`** — плоский JSON `{"key": "value"}`
- **`parseDotLang` / `rebuildDotLang`** — формат `key=value`, с поддержкой `#`-комментариев
- **`parseSnbt` / `rebuildSnbt`** — поиск по ключевым полям (`title`, `description`, `text`, ...)
- **`parseToml` / `rebuildToml`** — строковые значения, пропускает технические ключи (id, type, version, ...)
- **`parseCfg` / `rebuildCfg`** — значения в формате `S:key=value`
- **`parseNestedJson` / `rebuildNestedJson`** — рекурсивный обход вложенного JSON
- **`parseXml` / `rebuildXml`** — извлечение текста между тегами
- **`parsePlainText` / `rebuildPlainText`** — построчная обработка

Функция `hasTranslatableText()` фильтрует строки: должны содержать хотя бы одну латинскую букву и быть длиннее 1 символа.

## 🔐 Безопасность

- `DEEPL_API_KEY` хранится **исключительно** в `.env` на сервере
- Переменная **не имеет** префикс `NEXT_PUBLIC_` — браузер никогда её не получает
- Файл `.env` добавлен в `.gitignore` — ключ не попадёт в репозиторий
- Все запросы к DeepL API выполняются через Next.js API Routes на стороне сервера

## 🛠️ Устранение неполадок

### ❌ `DEEPL_API_KEY не задан`
- Проверь наличие файла `.env` в корне проекта
- Убедись, что ключ вставлен без лишних пробелов и кавычек
- Перезапусти сервер после изменения `.env` (`Ctrl+C` → `npm run dev`)

### ❌ `Нет английских lang файлов в JAR`
- Мод может не содержать локализацию вовсе
- Языковые файлы могут быть в нестандартной папке (не `assets/*/lang/`)
- Текст мода может быть встроен напрямую в код (`.class` файлы)

### ❌ Ошибка 429 (Too Many Requests)
- Превышен лимит запросов к DeepL API
- Система автоматически повторит попытку до 3 раз
- Если ошибка продолжается — проверь остаток символов на [deepl.com](https://www.deepl.com/account/usage)

### ❌ Файл не загружается
- Проверь размер файла (максимум **1000 MB**)
- Убедись, что расширение поддерживается (см. таблицу форматов)
- Открой консоль браузера (F12 → Console) для деталей ошибки

### ❌ Перевод пустой или неполный
- Часть строк могла быть пропущена фильтром `hasTranslatableText` (нет латиницы)
- Для SNBT: поддерживаются только стандартные ключи (`title`, `description`, `text`, и др.)
- Для TOML: технические ключи (`id`, `type`, `version`, ...) намеренно пропускаются
