# 📊 Отчеты о переводе - Руководство пользователя

## Что это?

Начиная с версии **3.10.0**, MOD_TRANSLATOR автоматически генерирует детальные отчеты о каждом переводе. Теперь вы можете увидеть **что именно было переведено** до скачивания файлов.

---

## Зачем это нужно?

### Проблемы, которые решает:

❌ **Раньше:**
- Непонятно что переводится в квестах
- Нет информации о переведенных строках в модах
- Невозможно проверить качество перевода
- Приходится скачивать и открывать файл чтобы увидеть результат

✅ **Теперь:**
- Видно все переведенные строки с примерами
- Понятно какие части квестов переводятся (title, description и т.д.)
- Можно проверить качество до скачивания
- Полная прозрачность процесса

---

## Как это работает?

### 1. Загрузите файл и запустите перевод

Как обычно:
1. Перетащите файл (.jar, .snbt, .json и т.д.)
2. Нажмите **"ЗАПУСТИТЬ ПЕРЕВОД"**
3. Дождитесь завершения

### 2. Откройте отчет

После завершения перевода появится новая секция:

```
// 04. ОТЧЕТ О ПЕРЕВОДЕ
▶ 📊 ОТЧЕТ О ПЕРЕВОДЕ                    [⬇ HTML]

Режим: Одиночный файл
Файлов: 1
Строк переведено: 150
Дата: 03.05.2026 18:15
```

Нажмите на **▶** чтобы развернуть отчет.

### 3. Просмотрите примеры переводов

В развернутом отчете вы увидите:

```
📄 quests/chapter1.snbt [SNBT]
   [SNBT] • 5 строк

   [snbt_line_3]
   EN: Complete the first quest
   RU: Завершите первый квест

   [snbt_line_7]
   EN: Gather 10 wood logs
   RU: Соберите 10 бревен

   ... и еще 3 строк(и)
```

### 4. Скачайте отчет (опционально)

Доступны два формата:

- **HTML отчет** - красивый, стилизованный, с подсветкой
- **Текстовый отчет** - для документации, простой текст

## Что показывает отчет?

### Для каждого файла:

1. **Имя файла** - полный путь (для JAR) или имя файла
2. **Формат** - JSON, LANG, SNBT, TOML, CFG, XML и т.д.
3. **Количество строк** - сколько строк было переведено
4. **Примеры переводов** - первые 10-20 строк с переводами

### Для каждой строки:

1. **Ключ** - идентификатор строки (если есть)
   - `item.minecraft.stone` - для предметов
   - `block.modname.ore` - для блоков
   - `snbt_line_5` - для квестов
   - `toml_line_8` - для конфигов

2. **Оригинал (EN)** - исходный английский текст
3. **Перевод (RU)** - переведенный русский текст

## Часто задаваемые вопросы

### Q: Отчет показывает все переводы?

**A:** Нет, в UI показываются первые 20 строк для каждого файла. Полный отчет можно скачать в HTML или текстовом формате.

### Q: Можно ли отключить отчеты?

**A:** Отчеты генерируются автоматически, но секция свернута по умолчанию. Просто не разворачивайте её, если не нужна.

### Q: Отчеты сохраняются?

**A:** Отчеты доступны только в текущей сессии. Если нужно сохранить - скачайте HTML или текстовый файл.

### Q: Что означают ключи типа "snbt_line_5"?

**A:** Это внутренние идентификаторы для форматов без явных ключей (SNBT, TOML, CFG). Они указывают на номер строки в файле.

### Q: Почему в квестах переводятся не все строки?

**A:** Переводятся только текстовые поля: `title`, `subtitle`, `description`, `text`, `name` и т.д. Технические поля (ID, типы, координаты) не переводятся.

### Q: Можно ли редактировать переводы в отчете?

**A:** Нет, отчет только для просмотра. Если нужно исправить перевод - отредактируйте скачанный файл вручную.

## Технические детали

### Форматы отчетов:

1. **JSON** (структурированный)
   - Для программного доступа
   - Содержит все данные
   - Используется внутри приложения

2. **TXT** (текстовый)
   - Для документации
   - Простой текст
   - Легко читается в любом редакторе

3. **HTML** (веб-страница)
   - Красивый дизайн
   - Стилизация в стиле терминала
   - Подсветка синтаксиса
   - Удобная навигация

### Что переводится в разных форматах:

| Формат | Что переводится | Пример ключа |
|--------|----------------|--------------|
| JSON (lang) | Все строковые значения | `item.minecraft.stone` |
| LANG | Все значения после `=` | `item.stone.name` |
| SNBT | title, description, text, name | `snbt_line_5` |
| TOML | Строковые значения (кроме технических) | `toml_line_8` |
| CFG | Строковые значения с пробелами | `cfg_line_12` |
| XML | Текст между тегами | `xml_3` |
| TXT | Все строки с буквами | `txt_0` |

## Примеры использования

### Сценарий 1: Проверка качества перевода квестов

1. Загрузите файл квеста `.snbt`
2. Запустите перевод
3. Откройте отчет
4. Проверьте примеры переводов
5. Если всё хорошо - скачайте файл
6. Если есть ошибки - отредактируйте вручную

### Сценарий 2: Документирование перевода мода

1. Переведите мод `.jar`
2. Откройте отчет
3. Скачайте HTML отчет
4. Приложите к релизу мода
5. Пользователи смогут увидеть что было переведено

### Сценарий 3: Отладка проблем с переводом

1. Файл переведен, но что-то не так
2. Откройте отчет
3. Найдите проблемную строку
4. Проверьте ключ и перевод
5. Исправьте вручную в скачанном файле
