# Решение проблемы с портами

## Проблема
При каждом запуске `npm run dev` сервер занимает новый порт (3000 → 3001 → 3002 → 3003...), потому что старые процессы не останавливаются.

## Решение

### Вариант 1: Использовать новый скрипт (Рекомендуется)

Вместо `npm run dev` используйте:

```bash
npm run dev:clean
```

Этот скрипт автоматически:
1. Останавливает все процессы на портах 3000-3003
2. Запускает сервер на порту 3000

### Вариант 2: Использовать bat-файл

Запустите файл `dev.bat` вместо `npm run dev`:

```bash
./dev.bat
```

### Вариант 3: Вручную остановить процессы

Если нужно вручную освободить порты:

```bash
powershell -Command "Get-NetTCPConnection -LocalPort 3000,3001,3002,3003 -ErrorAction SilentlyContinue | Select-Object -ExpandProperty OwningProcess -Unique | ForEach-Object { Stop-Process -Id $_ -Force -ErrorAction SilentlyContinue }"
```

Затем запустите:
```bash
npm run dev
```

## Почему это происходит?

Next.js dev server иногда не останавливается корректно при:
- Закрытии терминала
- Ctrl+C в терминале
- Перезапуске VSCode
- Ошибках в коде

Процессы остаются в памяти и занимают порты.

## Рекомендация

**Всегда используйте `npm run dev:clean`** - это гарантирует, что сервер запустится на порту 3000.
